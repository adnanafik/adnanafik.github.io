🧠 retro-pilot

PythonMITChromaDBLLM-as-judge

Autonomous incident post-mortem engine — reconstructs timelines, runs RCA, builds a learning knowledge base.

Hierarchical multi-agent system: an Orchestrator coordinates six specialists (evidence, timeline, RCA, action items, writer, evaluator). Every draft is scored by an LLM-as-judge against a 5-dimension rubric — low scores trigger a revision loop, bounded to 3 cycles to prevent runaway. Approved post-mortems are embedded and stored in ChromaDB so future incidents learn from past ones. Typed Pydantic contracts between every agent boundary.
